

input.scenario (required)
Select scenario for benchmark.

input.ssh_allowed_ips (required)
IPs to allow through SSH firewall. This field must be populated for any of benchmark automation to work! Using "*", "Any", an empty list and literal ranges "224.0.0.0/4", "255.255.255.255/32", "127.0.0.0/8", "169.254.0.0/16", "168.63.129.16/32" and "0.0.0.0/0" is not allowed.

input.client_id (null)
Azure Client ID to use.

input.client_secret (null)
Azure Service Principal secret to use.

input.custom_profile ("")
Select preprepared custom scenario for benchmark test.

input.dbOnly (false)
If set to "true", only the database VM is deployed.

input.expose_rdp (false)
Should RDP be exposed to the Internet?

input.global_tags ({})
Dictionary of tags that will be applied to deployment.

input.linux (true)
Run benchmark on Linux.

input.location ("West US")
What Azure region should be used for the resources.

input.make_key_vault (true)
If true, key vault will be created and keys uploaded into it.

input.rdp_allowed_ips (null)
IPs to allow through RDP firewall. By default allows no addresses. This value is only used when expose_rdp is set to true simultaneously. Using literal ranges"224.0.0.0/4", "255.255.255.255/32", "127.0.0.0/8", "169.254.0.0/16", "168.63.129.16/32" and "0.0.0.0/0" is not allowed.

input.resourcegroup ("IntelOptimizedCloudStack")
What resource group should be used to spawn the resources.

input.server_disks_data_size (512)
How big should each data disk be in GB? Minimally 1. Used in MSSQL, MySQL and Oracle (together with "spawn_disks"=true and mysql_mount_dir set to "/mnt/mysqldata" and oracle_mount_dir set to "/mnt/oracledata").

input.server_disks_wal_size (512)
Used in Oracle and MS SQL benchmark. Specify the disk size used for Oracle redo logs.

input.server_hammerdb_instancetype ("Standard_E32_v4")
Which instance type to use for HammerDB.

input.server_instancetype ("Standard_E64ds_v4")
Which instance type to use.

input.server_os_disksize (1024)
Size of database OS disk. Might need to be customized when not using spawn_disks set to "true" with large warehouses.

input.server_os_name ("OracleDB19")
Sets which operating will be used for the database virtual machine.

input.server_os_storagetype ("Standard")
Type of disks to use for Operating System. Currently supports Standard and Premium.

input.server_storagetype ("Premium")
Type of disks to use. Currently supports Standard and Premium.

input.spawn_disks (false)
Used in Oracle and MySQL benchmark. If set to false, creates DB on os disk. If set to true, spawns %server_disks_data_size% GB sized disk for database. It does not install the database on it unless respective paramter in ansible is set.

input.subscription_id (null)
Azure Subscription ID to use.

input.tenant_id (null)
Azure Tenant ID to use.

input.vm_tags ({})
Dictionary of additional tags that will be applied to VMs only.

input.windows (false)
Run benchmark on Windows.
